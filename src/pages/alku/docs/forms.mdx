# Forms

In Alku nearly all input controls have been harmonized to equal height. No width has been applied so you retain some
backwards compatibility with any existing HTML page that is unstyled or low on CSS.

## Form layouts

You can create form layouts with rather minimalistic syntax.


### Compact inline form

Here is a traditional web inline style login form with the help of `data-box` utility:

<figure role="presentation" style="padding-block: 16px 32px;">
	<form onsubmit="return false">
		<fieldset data-box="row">
			<legend>
				<strong>Sign in to continue</strong>
			</legend>
			<input type="email" placeholder="Email" />
			<input type="password" placeholder="Password" />
			<label for="default-remember">
				<input type="checkbox" id="default-remember" />
				<span>Remember me</span>
			</label>
			<button type="submit">Sign in</button>
		</fieldset>
	</form>
</figure>

### Complex forms

You can use `data-grid="auto"` to create fluid grids.

<figure role="presentation" style="background: var(--muted-02); padding-block: 48px;">
	<form data-column="mediumContent" onsubmit="return false">
		<fieldset data-grid="auto">
			<label>
				<span>First name:</span>
				<input name="firstname" />
			</label>
			<label>
				<span>Last name:</span>
				<input name="lastname" />
			</label>
			<label data--="fullRow">
				<span>Email:</span>
				<input aria-describedby="email-description" type="email" name="email" />
				<small aria-hidden="true" id="email-description" data-text-color="state">
					<span>Your email will never be exposed.</span>
				</small>
			</label>
			<button>Submit</button>
		</fieldset>
	</form>
</figure>

`<label />` works as form field which you can place to a grid.

## Field states

By specifying `aria-invalid` you can signal invalid and valid states.

Element following input that has `data-text-color="state"` will match the input field state color.

<figure role="presentation" style="background: var(--muted-02); padding-block: 48px;">
	<div data-column="mediumContent">
		<h3>Outside label</h3>
		<div data-grid="auto" style="--grid-auto: 10rem;">
			<input aria-invalid="true" value="Invalid" />
			<input aria-invalid="false" value="Valid" />
			<input readonly value="Read only" />
			<input disabled value="Disabled" />
		</div>
		<h3>Inside label</h3>
		<div data-grid="auto" style="--grid-auto: 10rem;">
			<label>
				<span>Invalid:</span>
				<input aria-describedby="invalid-desc" aria-invalid="true" />
				<small aria-hidden="true" id="invalid-desc" data-text-color="state">
					<span>Error happened</span>
				</small>
			</label>
			<label>
				<span>Valid:</span>
				<input aria-describedby="valid-desc" aria-invalid="false" />
				<small aria-hidden="true" id="valid-desc" data-text-color="state">
					<span>Submit ok</span>
				</small>
			</label>
			<label>
				<span>Read only:</span>
				<input aria-describedby="readonly-desc" readonly />
				<small aria-hidden="true" id="readonly-desc" data-text-color="state">
					<span>Copy if needed</span>
				</small>
			</label>
			<label>
				<span>Disabled:</span>
				<input aria-describedby="disabled-desc" disabled />
				<small aria-hidden="true" id="disabled-desc" data-text-color="state">
					<span>Maybe don't use</span>
				</small>
			</label>
		</div>
	</div>
</figure>

Usually you should avoid disabled controls as they do not really provide any value to user.

You should assign invalid and valid states after blur and/or form submit.

```html
<div data-grid="auto" style="--grid-auto: 10rem;">
	<input aria-invalid="true" value="Invalid" />
	<input aria-invalid="false" value="Valid" />
	<input readonly value="Read only" />
	<input disabled value="Disabled" />
</div>
```
